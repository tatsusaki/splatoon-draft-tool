# 機能追加用テンプレート

このファイルは、新しい機能を追加したいときに、
いきなり実装するのではなく「設計 → 実装」の順で進めるための指針です。

## 基本方針

- まず「どんな機能が欲しいか」をテキストで整理する
- 既存構造のどこに組み込むかを一緒に考える
- 影響範囲を確認してから実装案を出す
- コードはできるだけ既存のレイヤー分け（A〜D）に沿って追加する

## ユーザーが依頼するときのテンプレ

---

FEAT: 新しい機能を追加したいです。

欲しい機能：
- （例）観戦回数が少ない人を優先的に観戦に回すオプション機能

現状の問題や動機：
- （例）同じ人が観戦になりがちなので、もう少し公平にしたい

ざっくりしたイメージ：
- （例）チェックボックスで「観戦回数を考慮する」をONにしたら有効になる

---

## ChatGPT の進め方

1. まずテキストベースで仕様案を整理・提案する
2. 既存コードのどの辺に組み込むのが自然かを検討する
3. 影響範囲を説明したうえで、実装の方針を提示する
4. 必要に応じて、実装に使う関数やデータ構造を定義する
5. 最後に、実装コード案を提示する
